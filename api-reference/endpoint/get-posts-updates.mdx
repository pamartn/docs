---

## Endpoint

```
GET /api/v1/posts/updates
```

## Authentication

Include your API key in the request header:

```bash
X-API-Key: your-api-key-here
```

## Credits

This endpoint consumes **1 credit per 1-15 profiles** (rounded up):

- 1-15 profiles = 1 credit
- 16-30 profiles = 2 credits
- 31-45 profiles = 3 credits
- 46-60 profiles = 4 credits

## Parameters

<ParamField query="profileUrlsOrUrns" type="string" required>
  Comma-separated list of LinkedIn profile URLs or URNs (maximum 60 profiles).
  
  Examples:
  - `https://www.linkedin.com/in/username1,https://www.linkedin.com/in/username2`
  - `urn:li:fsd_profile:ACoAAABCDEF,https://www.linkedin.com/in/username`
</ParamField>

<ParamField query="count" type="integer" default="15">
  Number of posts to return PER PROFILE (not total across all profiles).
  
  - Minimum: 1
  - Maximum: 100
  - Default: 15
  
  **Example:** With 2 profiles and `count=15`, you will receive up to 30 posts total (15 from each profile).
</ParamField>

## Response

<ResponseField name="posts" type="array">
  Array of recent LinkedIn posts aggregated from all specified profiles

<Expandable title="Post object">
  <ResponseField name="urn" type="string">
    Unique post identifier
  </ResponseField>
  
  <ResponseField name="text" type="string">
    Post content/text
  </ResponseField>
  
  <ResponseField name="publishedAt" type="string">
    ISO 8601 timestamp of when the post was published
  </ResponseField>
  
  <ResponseField name="numberOfLikes" type="integer">
    Total likes count
  </ResponseField>
  
  <ResponseField name="numberOfComments" type="integer">
    Total comments count
  </ResponseField>
  
  <ResponseField name="numberOfReposts" type="integer">
    Total reposts/shares count
  </ResponseField>
  
  <ResponseField name="imageUrl" type="string">
    URL of post image (if present)
  </ResponseField>
  
  <ResponseField name="videoUrl" type="string">
    URL of post video (if present)
  </ResponseField>
  
  <ResponseField name="carouselPdfUrl" type="string">
    URL of PDF document (if the post is a carousel/document post)
  </ResponseField>
  
  <ResponseField name="profileUrn" type="string">
    LinkedIn profile URN of the post author. This field is especially useful for differentiating posts from different profiles in the aggregated response.
  </ResponseField>
</Expandable>
</ResponseField>

## Example Request

<CodeGroup>

```bash cURL
curl -X GET "https://api.fetchin.io/api/v1/posts/updates?profileUrlsOrUrns=https://www.linkedin.com/in/williamhgates,https://www.linkedin.com/in/jeffweiner08&count=15" \
  -H "X-API-Key: your-api-key-here"
```

```javascript JavaScript
const profiles = [
  'https://www.linkedin.com/in/williamhgates',
  'https://www.linkedin.com/in/jeffweiner08'
].join(',');

const response = await fetch(
  `https://api.fetchin.io/api/v1/posts/updates?profileUrlsOrUrns=${encodeURIComponent(profiles)}&count=15`,
  {
    headers: {
      'X-API-Key': 'your-api-key-here'
    }
  }
);

const data = await response.json();
console.log(data.posts);
```

```python Python
import requests

headers = {
    'X-API-Key': 'your-api-key-here'
}

profiles = 'https://www.linkedin.com/in/williamhgates,https://www.linkedin.com/in/jeffweiner08'

params = {
    'profileUrlsOrUrns': profiles,
    'count': 15
}

response = requests.get(
    'https://api.fetchin.io/api/v1/posts/updates',
    headers=headers,
    params=params
)

data = response.json()
print(data['posts'])
```

```php PHP
<?php

$apiKey = 'your-api-key-here';
$profiles = 'https://www.linkedin.com/in/williamhgates,https://www.linkedin.com/in/jeffweiner08';
$count = 15;

$url = "https://api.fetchin.io/api/v1/posts/updates?profileUrlsOrUrns=" . urlencode($profiles) . "&count=" . $count;

$ch = curl_init();
curl_setopt($ch, CURLOPT_URL, $url);
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
curl_setopt($ch, CURLOPT_HTTPHEADER, [
    'X-API-Key: ' . $apiKey
]);

$response = curl_exec($ch);
$data = json_decode($response, true);

curl_close($ch);
print_r($data['posts']);
?>
```

## Notes

<Info>
  This endpoint consumes 1 credit per 1-15 profiles (rounded up). The cost scales with the number of profiles requested.
</Info>

<Warning>
  Posts are returned in reverse chronological order (newest first), aggregated from all specified profiles. The `count` parameter specifies posts PER PROFILE, so 3 profiles with count=15 returns up to 45 posts total.
</Warning>

<Tip>
  This endpoint scrapes each profile individually using reliable methods, ensuring consistent and deterministic results.
</Tip>

<Tip>
  You can mix profile URLs and URNs in the same request. Both formats are supported.
</Tip>
