## Endpoint

```
GET /api/v1/profile/connections
```

## Authentication

Include your API key in the request header:

```bash
X-API-Key: your-api-key-here
```

## Parameters

<ParamField query="profileUrlOrUrn" type="string" required>
  LinkedIn profile URL or URN.
  
  Examples:
  - `https://www.linkedin.com/in/username`
  - `urn:li:member:123456789`
</ParamField>

<ParamField query="count" type="integer" default="10">
  Number of connections to fetch per page.
  
  - Minimum: 1
  - Maximum: 10
  - Default: 10
</ParamField>

<ParamField query="start" type="integer" default="0">
  Offset for pagination. Increment by `count` (default 10) to get the next page.
  
  - Default: 0
</ParamField>

## Response

<ResponseField name="connections" type="array">
  Array of connection profiles

<Expandable title="Connection object">
  <ResponseField name="name" type="string">
    Full name of the connection
  </ResponseField>
  
  <ResponseField name="profileUrl" type="string">
    LinkedIn profile URL
  </ResponseField>
  
  <ResponseField name="profilePictureUrl" type="string">
    Profile picture URL (may be absent if the user has no profile picture)
  </ResponseField>
  
  <ResponseField name="headline" type="string">
    Professional headline
  </ResponseField>
  
  <ResponseField name="location" type="string">
    Geographic location
  </ResponseField>
</Expandable>
</ResponseField>

<ResponseField name="hasMore" type="boolean">
  Indicates whether there are more connections to fetch.
</ResponseField>

## Pagination

This endpoint uses offset-based pagination via the `start` parameter. To fetch all connections:

1. Make an initial request with `start=0`
2. If `hasMore` is `true`, increment `start` by `count` (default 10) and make another request
3. Continue until `hasMore` is `false`

```javascript
async function fetchAllConnections(profileUrl) {
  const allConnections = [];
  let start = 0;
  const count = 10;
  let hasMore = true;

  while (hasMore) {
    const params = new URLSearchParams({
      profileUrlOrUrn: profileUrl,
      count: String(count),
      start: String(start)
    });

    const response = await fetch(
      `https://api.fetchin.io/api/v1/profile/connections?${params}`,
      { headers: { 'X-API-Key': 'your-api-key-here' } }
    );

    const data = await response.json();
    allConnections.push(...data.connections);
    hasMore = data.hasMore;
    start += count;
  }

  return allConnections;
}
```

## Notes

<Info>
  This endpoint consumes 1 credit from your monthly quota per API call, regardless of the `count` parameter value. Failed requests (errors) do not count against your quota.
</Info>

<Warning>
  LinkedIn limits connections results to 10 per page. The `count` parameter can be used to request fewer results, but cannot exceed 10.
</Warning>
