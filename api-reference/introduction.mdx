## Base URL

```
https://your-domain.com/api/v1
```

Replace `your-domain.com` with your actual Fetchin deployment domain.

## Authentication

All API requests must include your API key in the `X-API-Key` header:

```bash
X-API-Key: your-api-key-here
```

## Response Format

All successful responses return JSON data. Error responses include an `error` field:

### Success Response

```json
{
  // ... response data
}
```

### Error Response

```json
{
  "error": "Error message"
}
```

## Common Parameters

### profileUrlOrUrn

LinkedIn profile identifier. Can be either:

- **Profile URL**: `https://www.linkedin.com/in/username`
- **Profile URN**: `urn:li:member:123456789`

```bash
?profileUrlOrUrn=https://www.linkedin.com/in/williamhgates
```

## Rate Limiting

Fetchin implements two types of rate limiting:

- **Per-second**: Maximum 10 requests per second
- **Monthly quota**: Based on your plan (100 for Trial, custom for Enterprise)

See [Rate Limits](/concepts/rate-limits) for detailed information and best practices.

## Error Codes

| Code | Description |
|------|-------------|
| 200  | Success |
| 400  | Bad Request - Invalid parameters |
| 401  | Unauthorized - Invalid or missing API key |
| 429  | Too Many Requests - Quota exceeded |
| 500  | Internal Server Error |
